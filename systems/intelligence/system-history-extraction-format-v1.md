# THE MUSCLE PT — SYSTEM HISTORY EXTRACTION FORMAT v1

## Purpose

This document defines how founder thinking is captured and converted into system intelligence.

It ensures that:
- raw conversation is preserved
- useful signal is extracted
- patterns are identified
- insights are integrated into the system

---

## Core Principle

Raw conversation is memory.

Extracted structure is intelligence.

The system must always prefer:
- structured entries
over:
- raw transcripts

---

## 1. INPUT SOURCE

Primary input comes from:

- verbal thinking
- conversational reasoning
- chat-based exploration

This input is:
- high value
- unstructured
- inconsistent

---

## 2. STORAGE MODEL

Raw input is stored separately from processed intelligence.

### Raw Storage

records/system-history/raw/

Contains:
- full transcripts
- exported chat conversations
- original, unmodified input

Example:
founder-claude-conversation-archive.md

---

### Processed Storage

records/system-history/extracted/

Contains:
- cleaned entries
- structured insights
- classified signals

---

### Pattern Storage

records/system-history/patterns/

Contains:
- repeated ideas
- recurring decisions
- behavioral patterns

---

## 3. EXTRACTION FLOW

Capture  
→ Clean  
→ Segment  
→ Classify  
→ Interpret  
→ Map  
→ Store  

---

## 4. CLASSIFICATION TYPES

Each extracted entry must be labeled as:

- Decision  
- Insight  
- Pattern  
- Friction  
- Hypothesis  
- Noise  

---

## 5. STRUCTURED ENTRY FORMAT

Each entry must include:

- entry_id  
- date_recorded  
- context  

- raw_statement  
- classification  
- interpretation  

- system_mapping  
- actionability  
- confidence_level  

---

## 6. SYSTEM MAPPING

Each entry must map to one or more:

- execution-playbook  
- movement-case-engine  
- content-case-flywheel  
- governance layer  
- research layer  

---

## 7. USAGE RULES

Always:
- use extracted entries first  
- use patterns second  
- use raw archive only when needed  

Never:
- rely on raw conversation as primary logic  
- skip extraction  
- over-interpret input  

---

## 8. PATTERN PROMOTION

Repeated entries → pattern  
Repeated patterns → system input  

---

## 9. FAILURE MODES

- storing raw data without extraction  
- over-analyzing simple input  
- misclassifying signals  
- not integrating outputs  

---

## 10. FINAL DEFINITION

This layer converts:

thinking → structured intelligence  

It ensures:
- nothing valuable is lost  
- patterns are recognized  
- the system evolves from real behavior  

---

## SYSTEM SUMMARY

Capture → Extract → Structure → Map → Store → Use
