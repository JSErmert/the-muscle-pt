# THE MUSCLE PT — EXECUTION PLAYBOOK v1

## Purpose

This document defines how work is executed on a daily and weekly basis.

It translates the system into:
- clear actions
- repeatable workflows
- minimal decision-making during execution

The goal is to ensure:
- consistent output
- low cognitive load
- reliable system usage

---

## Core Principle

Do the work first.

Do not overthink.

Use the system to guide, not slow down.

---

## 1. DAILY EXECUTION FLOW

### Sessions

During client sessions:

- assess movement
- identify case type
- determine root cause
- apply intervention
- observe response

Do not over-structure.

Just be aware of:
- case
- root cause
- intervention
- outcome

---

### After Sessions

Immediately after:

- identify outcome:
  - successful
  - failed
  - inconclusive

- capture:
  - what worked
  - what did not work
  - key observation

Keep it short.

---

### End of Day

Record:

- one key insight
- one pattern
- one notable case

This is not a report.

This is signal capture.

---

## 2. CONTENT EXECUTION FLOW

### Content Creation

Instead of asking:

What should I post?

Ask:

What did I observe?

---

### Content Workflow

1. Select insight from real-world observation  
2. Identify content bucket  
3. Choose hook format  
4. Write script  
5. Write caption  
6. Film and edit  
7. Post  
8. Engage with audience  

---

### Posting Rules

- prioritize clarity over perfection
- prioritize real insight over invention
- do not post low-quality content to maintain frequency

---

## 3. WEEKLY EXECUTION FLOW

### Weekly Review

Once per week:

- review cases
- identify repeated patterns
- identify repeated failures
- identify best-performing content

---

### Weekly Planning

- select 3–4 content topics based on:
  - repeated insights
  - strong signals
  - audience relevance

- schedule content

---

## 4. SYSTEM USAGE RULES

- do not open multiple system documents during execution
- do not over-reference rules
- trust the workflow
- capture only what matters

---

## 5. PRIORITY RULE

If unsure what to do:

- follow the queue engine
- work on highest-priority item

---

## 6. FAILURE RULE

If something feels unclear:

- do not force clarity
- mark as inconclusive
- gather more data

---

## 7. SIMPLICITY RULE

Execution must remain:

- fast
- intuitive
- repeatable

If it feels complex:

Simplify immediately.

---

## 8. FINAL DEFINITION

This playbook ensures:

- work gets done
- thinking is structured
- output is consistent
- system is actually used

---

## SYSTEM SUMMARY

Run sessions  
Capture outcomes  
Extract insights  
Create content  
Repeat daily
